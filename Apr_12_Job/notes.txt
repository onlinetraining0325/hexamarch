Spring Boot with Spring Batch 

Spring batch is a lightweight yet robust framework designed for batch processing, the automated execution of large data tasks without human intervention. It provides reusable components for logging, transaction management, job scheduling and error handling.

Batch processing refers to executing repetitive, data-intensive tasks in bulk.  

	-> Processing large datasets
	-> Data Migration
	-> Generating Reports
	-> ETL (Extract, Transform and Load) Operations

Spring Batch is purpose-built for such use cases by splitting jobs into smaller, manageable steps that can run parallelly.

	A job in spring batch represents the complete batch process, while steps define the logical phases within that job.

	Job : Encapsulates the full batch process, consisting of multiple steps
	Step : Represents one stage of job - typically involves reading, processing and writing 		data

	Flow : Defines the execution order of steps, you can create conditional or parallel flows
